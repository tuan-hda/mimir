6. Describe a time you improved performance.
   - Sample answer: One performance story is the cursor pagination migration. Offset pagination was simple, but as datasets grew it became slower and less stable when records changed. I helped propose a cursor-based design using stable ordering, aligned it with backend engineers, and updated frontend flows around next/previous cursors instead of page numbers. On the frontend, that meant cache keys, loading states, duplicate prevention, and URL behavior had to be reconsidered. The result was a more scalable pattern with better response time for large operational lists. The lesson I took from it is that performance work should start from user-visible pain and be verified with measurement, not guessed.
